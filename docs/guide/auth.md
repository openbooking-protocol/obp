# Authentication

OBP uses two authentication methods:

| Method | Header | Used for |
|---|---|---|
| API Key | `X-Api-Key: obpk_...` | Provider management (server-side) |
| Bearer token | `Authorization: Bearer <jwt>` | OAuth2 for end-user flows |
| None | — | Public endpoints (discovery, services, slots, booking status) |

## API Keys

API keys are the primary auth method for providers managing their data.

### Generating an API key

API keys are generated by server admins:

```bash
curl -X POST https://obp.example.com/obp/v1/auth/keys \
  -H "X-Api-Key: ADMIN_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "provider_id": "prv_abc123",
    "scopes": ["read", "write"]
  }'
```

Response:
```json
{
  "id": "key_xyz",
  "key": "obpk_abc123...secret",
  "provider_id": "prv_abc123",
  "scopes": ["read", "write"],
  "created_at": "2026-04-01T10:00:00Z"
}
```

::: warning Store securely
The full API key is only shown once. Store it in a secrets manager or environment variable.
:::

### Using an API key

Include the key in the `X-Api-Key` header:

```bash
curl https://obp.example.com/obp/v1/bookings \
  -H "X-Api-Key: obpk_abc123...secret"
```

### API key scopes

| Scope | Description |
|---|---|
| `read` | Read provider data, bookings, analytics |
| `write` | Create/update services, manage bookings |
| `admin` | Server administration |

### Rotating an API key

```bash
curl -X POST https://obp.example.com/obp/v1/auth/keys/key_xyz/rotate \
  -H "X-Api-Key: obpk_abc123...secret"
```

## OAuth2 (for frontend apps)

For browser-based apps that act on behalf of users, OBP supports OAuth2 Authorization Code flow with PKCE.

### Flow overview

```
1. User clicks "Login with OBP"
2. App redirects to /oauth/authorize with PKCE challenge
3. User authenticates and approves scopes
4. Server redirects back with authorization code
5. App exchanges code for access + refresh tokens
6. App uses access token in Bearer header
```

### Authorization request

```
GET /oauth/authorize
  ?client_id=your_client_id
  &redirect_uri=https://yourapp.com/callback
  &response_type=code
  &scope=bookings:read bookings:write
  &code_challenge=<sha256-of-verifier>
  &code_challenge_method=S256
  &state=<random>
```

### Token exchange

```bash
curl -X POST https://obp.example.com/oauth/token \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "grant_type=authorization_code
      &code=AUTH_CODE
      &redirect_uri=https://yourapp.com/callback
      &client_id=your_client_id
      &code_verifier=PKCE_VERIFIER"
```

Response:
```json
{
  "access_token": "eyJ...",
  "refresh_token": "eyJ...",
  "token_type": "Bearer",
  "expires_in": 3600
}
```

### Using a Bearer token

```bash
curl https://obp.example.com/obp/v1/bookings \
  -H "Authorization: Bearer eyJ..."
```

## Public endpoints

These endpoints require no authentication:

- `GET /.well-known/obp`
- `GET /obp/v1/providers`
- `GET /obp/v1/providers/:id`
- `GET /obp/v1/categories`
- `GET /obp/v1/services`
- `GET /obp/v1/services/:id`
- `GET /obp/v1/slots`
- `POST /obp/v1/slots/:id/hold`
- `POST /obp/v1/bookings`
- `GET /obp/v1/bookings/:id`
- `POST /obp/v1/bookings/:id/cancel`
